# 🤖 GFS-X Sim-to-Real RL Agent

Этот проект посвящен управлению реальным гусеничным роботом XiaoR Geek GFS-X с использованием обучения с подкреплением (Reinforcement Learning) в архитектуре **Hardware-In-the-Loop (HIL)**. 

Обучение агента (PPO + LSTM) происходит в симуляторе Unity ML-Agents, после чего обученная модель (`.onnx`) управляет физическим роботом через ROS-мост, принимая данные с реальных датчиков в режиме реального времени.

## 🛠 Архитектура проекта

Проект разделен на следующие основные директории:

1. **Unity Environment (`/UnityProject`)**: Цифровой испытательный полигон. Содержит 3D-модель робота, скрипты `VirtualSensors`, ROS TCP Connector и логику начисления наград (Curriculum Learning) для обучения нейросети.
2. **ROS / Raspberry Pi (`/XiaoRGeek_GFS-x`)**: Аппаратная часть на базе RPi 4B. Здесь хранятся референсные скрипты с реального робота: логика чтения аппаратных ИК-датчиков, обработка команд от нейросети и управление ШИМ-сигналами дифференциального привода.
3. **Компьютерное зрение (`/Yolo`)**: Модуль детекции объектов. Содержит Python-скрипты инференса YOLO и сами веса моделей для распознавания целевого объекта (мяча) и расчета его координат в кадре.
4. **Обучение агента (`/Learning`)**: Конфигурации и веса нейросетей. Включает в себя гиперпараметры ML-Agents (`config.yaml`), логи предыдущих тренировок для TensorBoard и итоговые обученные модели в формате `.onnx`.
5. **Документация (`/Docs`)**: Техническая база. Сборник официальных мануалов по сборке, распиновке и настройке платформы XiaoR Geek GFS-X.

## 🚀 Ключевые особенности
* **Асинхронное HIL-управление:** Отказ от пространственной синхронизации цифрового двойника ради устранения проблемы счисления пути (Dead Reckoning Drift).
* **Sensor Fusion:** Объединение данных ультразвукового дальномера, ИК-сенсоров и компьютерного зрения в единый вектор наблюдений размерностью 10.
* **Deadzone Mapping:** Программная адаптация линейных выходов нейросети к физическому трению покоя коллекторных двигателей.
